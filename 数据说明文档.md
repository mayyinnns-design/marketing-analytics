# 营销活动数据分析平台 - 数据说明文档

## 📋 数据表格说明

### 表格1：公司购买明细

**用途**：记录每家企业在各个活动中的购买行为和订单金额

**必需字段**：
- **活动名称**：营销活动的名称
- **公司名称**：购买企业的名称
- **产品名称**：购买的产品名称
- **合同订单金额/万元**：总的合同订单金额（单位：万元）
- **新购合同订单金额/万元**：新购买的合同金额（单位：万元）
- **新客户合同订单金额/万元**：新客户产生的合同金额（单位：万元）

**示例**：
```
活动名称,公司名称,产品名称,合同订单金额/万元,新购合同订单金额/万元,新客户合同订单金额/万元
新年营销峰会,腾讯科技,企业版CRM,50,50,0
新年营销峰会,阿里巴巴,高级咨询服务,80,80,80
```

### 表格2：活动线索数据

**用途**：记录每个活动的整体效果指标

**必需字段**：
- **活动名称**：营销活动的名称
- **线索数**：剔除无效后的线索数量
- **触达企业数**：活动触达的企业去重后数量
- **合同订单转化率**：有合同订单的企业数占触达企业数的比例（%）
- **合同订单金额/万元**：该活动产生的总合同金额（单位：万元）
- **新购合同订单转化率**：新购合同订单企业数占触达企业数的比例（%）
- **新客户合同订单金额/万元**：新客户产生的合同金额（单位：万元）

**示例**：
```
活动名称,线索数,触达企业数,合同订单转化率,合同订单金额/万元,新购合同订单转化率,新客户合同订单金额/万元
新年营销峰会,580,48,6.25,160,6.25,110
数字化转型论坛,420,35,8.57,140,8.57,95
```

## 📊 核心指标说明

### 1. 线索相关指标
- **线索数**：剔除友商公司、腾讯员工、测试数据等无效线索后的有效线索数量
- **触达企业数**：参与活动的企业数量（去重后）
- **单个线索价值**：合同订单金额 ÷ 线索数，衡量每个线索的平均价值

### 2. 转化率指标
- **合同订单转化率**：产生购买的企业数 ÷ 触达企业数 × 100%
- **新购合同订单转化率**：产生新购的企业数 ÷ 触达企业数 × 100%

### 3. 金额指标
- **合同订单金额**：该活动产生的所有订单总金额
- **新购合同订单金额**：首次购买或购买新产品的订单金额
- **新客户合同订单金额**：新客户（首次接触）产生的订单金额
- **客均订单金额**：合同订单金额 ÷ 购买企业数

### 4. 染色逻辑说明
根据您提供的业务规则，线索从活动创建日起，后续6个月内产生的所有购买都会被计入该活动的转化数据中。

**举例**：
- 1月活动触达了A公司
- A公司在3月和5月分别签订了合同
- 如果3月和5月也有活动触达了A公司，那么：
  - 1月活动的数据包含：A公司3月和5月的订单
  - 3月活动的数据包含：A公司3月和5月的订单
  - 5月活动的数据包含：A公司5月的订单

这种"染色逻辑"确保了多触点营销的效果都能被正确归因。

## 🎯 系统提供的分析维度

### 1. 营销概览
- 总销售额、总线索数、触达企业数
- 平均转化率、单个线索价值
- 最高消费企业、最佳活动、最畅销产品

### 2. 企业购买分析
- 各企业的总消费、新购金额、新客户金额
- 参与活动数、购买产品种类、购买次数
- 企业消费排名

### 3. 活动效果分析
- 各活动的线索数、触达企业数、转化率
- 销售额、客单价、单个线索价值
- 活动效果对比（按销售额、转化率、线索量等维度）

### 4. 产品销售分析
- 各产品的总销售额、新购金额、新客户金额
- 购买次数、购买企业数、相关活动数
- 产品平均单价、产品排名

### 5. 智能查询
- 按企业查询：查看该企业的所有活动参与记录和购买明细
- 按活动查询：查看该活动的所有效果数据和购买详情
- 按产品查询：查看该产品的所有销售数据和相关企业/活动

## 💡 数据质量建议

### 1. 数据完整性
- 确保所有必需字段都有值
- 企业名称、活动名称需保持一致（同一实体使用同一名称）
- 金额数据使用数值格式，不要包含货币符号

### 2. 数据准确性
- 线索数应为去除无效线索后的数量
- 转化率可以是百分数（如 8.5%）或小数（如 0.085）
- 金额统一使用万元为单位

### 3. 数据关联
- 公司购买明细中的活动名称应与活动线索数据中的活动名称一致
- 建议同时上传两个表格以获得最全面的分析

## 📈 系统还能分析出的其他结论

基于您提供的数据，系统自动计算并展示以下深度指标：

### 营销效率指标
1. **ROI相关**
   - 单个线索价值 = 总销售额 ÷ 总线索数
   - 获客成本分析（如果有成本数据）
   - 客均订单价值 = 总销售额 ÷ 购买企业数

2. **转化漏斗**
   - 线索 → 触达企业 → 购买企业 的转化路径
   - 新客户转化率与老客户复购率对比
   - 各活动的转化效率排名

3. **客户价值分析**
   - 高价值客户识别（消费金额Top企业）
   - 客户活跃度（参与活动次数）
   - 客户购买行为分析（购买产品种类、频次）

4. **活动效果评估**
   - 最佳活动识别（综合转化率和销售额）
   - 高转化率活动分析
   - 高线索量活动的转化能力

5. **产品策略洞察**
   - 明星产品识别（高销售额+高购买次数）
   - 产品与活动的最佳组合
   - 产品客户群体分析

6. **新客户获取分析**
   - 新客户占比
   - 新客户平均订单价值
   - 最适合新客户获取的活动/产品

## ❓ 常见问题

**Q: 必须同时上传两个表格吗？**
A: 不是必须的。只上传购买明细也可以分析，但上传线索数据能获得更全面的转化率等指标。

**Q: 转化率应该填百分数还是小数？**
A: 两种都支持。系统会自动识别（如 8.5% 或 0.085 都可以）。

**Q: 金额字段必须是万元吗？**
A: 字段名称中注明了"/万元"的需要使用万元，系统会按此单位展示结果。

**Q: 一个企业参加多个活动如何记录？**
A: 在购买明细中，每个活动×企业×产品的组合占一行。同一企业的不同活动记录为多行。

**Q: 如何理解"染色逻辑"？**
A: 企业在某个活动后6个月内的所有购买，都会被计入该活动的转化数据，即使中间还参加了其他活动。
